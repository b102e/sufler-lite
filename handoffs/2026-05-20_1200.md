## Known Problems

1. **Filler timer delay** — `callSuggestAPI` shows filler phrase after 1600ms; the filler may not clear if `res.body` is null (early return). Fixed with `clearFiller()` before early return.
2. **Duplicate transcript** — If Deepgram sends a final right at listening start (race with stopped=false), may get stale text. Mitigated but edge case exists.
3. **readMode not in CallProfile** — readMode lives in sessionStorage("sufler:readMode"), not passed through CallProfile. If sessionStorage is cleared between onboarding and call, defaults to "italian".
4. **CallSummaryReview.tsx** — kept but removed from flow. Dead code but harmless.
5. **Several live-call components deleted** — `CallInstructionModal`, `LiveCallControls`, `LiveTranscriptPanel`, `ResponseSuggestionGrid`, `ListeningIndicator` all removed. Do not restore.

## Architecture Decisions

- **No SQLite**: All data ephemeral. Transcript lives in React state → sessionStorage(sufler:result) → deleted on read.
- **No grace period**: Deepgram creates fresh connection per listening cycle. Prevents stale transcripts crossing cycles.
- **Streaming suggest**: Text format `ITALIAN:/RUSSIAN:/TRANSLIT:/FAREWELL:` lines, parsed via regex as stream arrives.
- **uiMessages as single source of truth**: All bubble types (user/assistant/reading-question/ready-message/instruction) in one array to preserve insertion order.
- **Skip review screen**: onComplete(profile) from ClarificationChat → new/page.tsx creates session synchronously → checkMicAndProceed immediately.
- **Binary WebSocket audio**: Removed base64 encoding. isBinary flag distinguishes control messages (JSON text) from audio (binary).

## Next Recommended Step

**Persist readMode across sessions** — currently sessionStorage("sufler:readMode") is set during onboarding and read at call mount. If user clears storage or opens from another device, defaults to "italian". Consider:
1. Add readMode to the sessionStorage session object: `session:${sessionId}` already written in new/page.tsx
2. Read it in call/[id]/page.tsx from the session object, not separately

This ensures readMode is bundled with the session data and survives the same lifecycle.
